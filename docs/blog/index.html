§01
THE PROBLEM

资产，比你以为的多得多

这些数据是你和 Claude 之间的工作语言。它们让 Claude 知道你的代码风格、团队的术语、你昨天告诉它"这个测试不要再忘了跑"。它们不是配置，是记忆。

风控收紧的时候，你要么换号、要么换 agent。换 agent 这条路上，绝大多数迁移工具只搬 CLAUDE.md——其他全部静默丢失。会话内容、子 agent 轨迹、tool-result 侧车、shell 快照、企业策略 CLAUDE.md、.claude/commands/ 里你自己写的 slash 命令……这些是你真正用得上的东西。

不是配置，是记忆。
不是文件，是工作语言。

claude-code-migration 想解决的事，就是让这些资产在迁移时活下来。

§02
ARCHITECTURE

不是点对点，是中间层

Claude Code、Cursor、OpenCode、Hermes Agent、Windsurf——每个 agent 有自己的本地数据格式。如果做点对点迁移，N 个源 × M 个目标 = N×M 个适配器。新增一个平台就要写 N+M 个新对子。

我们做了 N+M：所有源 → 一个中间表示（Workspace Dossier；代码里叫 CanonicalData）→ 所有目标。

dossier.json 是你拥有的：纯 JSON、可 diff、可 commit、可加密备份。哪天再想换 agent，再 apply 一次就行。它不绑任何厂商。

新加一个 Agent 平台，只需要写一个 parser 或 writer，自动获得它和其他所有 Agent 的互通能力。

§03
THREE STEPS

对齐你的心智模型

三步——指项目、导出档案、生成目标。每一步都是独立的命令，每一步都有可审计的产出。

i.
point

告诉工具
项目在哪

给它一个 --project 路径。工具会自动定位 Claude 本地数据，包括项目 .claude/ 和 ~/.claude/。

--project /path/to/proj
ii.
export

识别+导出为
Workspace Dossier

扫描 60+ 种数据类型、归一到 CanonicalData、敏感数据自动 redact、写到磁盘 0o600 权限。

ccm export --out dossier.json
iii.
apply

生成
目标 Agent 项目

同一份档案，反复 apply 到不同 target。换 agent 不必重扫——大项目的 session JSONL 常常几十 MB。

ccm apply --target hermes

想一把梭？ ccm migrate --project . --target hermes,opencode,cursor,windsurf 把 export + apply 串起来。

§04
SAFETY DEFAULTS

安全是默认，不是选项

dossier.json 写盘自动完成下面这些事。每一项都是默认行为，不需要你打开开关。

  • 明文密钥 redact — MCP headers 里的 Bearer、*_API_KEY、会话正文里粘贴的 sk-ant-* / ghp_* / AKIA* / PEM 私钥 / BigModel 32hex.16alnum 全替换成 ${CC_<PATH>} 占位符。adapter 把占位符原样传到目标 config，运行时读 env。
  • 0o600 文件权限 — user-only。共享机器上其他账户 cat 不到。
  • secrets-manifest 审计文件 — 同目录生成，只含 SHA256 前缀和建议的 env var 名，不含原文。可以审计"这次脱敏了哪些"。
  • 不动你的真实项目 — 默认所有产物先 staging 到 <out>/<target>-target/<name>/，确认无误才用 --in-place，且必须在干净 git 分支上。

§06
ALWAYS-ON

Hub 模式 · v1.1+

上面的三步是一次性迁移。如果你想让多个 agent 持续共享同一份记忆——Claude Code 改了一段，Cursor 这边马上读得到——v1.1 加了 hub 模式。 

pip install 'claude-code-migration[hub]'

ccm hub init                     # 本地 SQLite buffer
ccm hub serve --remote           # captures → outbox → Supabase
ccm hub mcp-serve                # 通过 MCP 让任何 agent 读 L4 mirror

四层架构：L1+L2 Supabase（pgvector + tsvector）· L3 hub-agent 守护进程 · L4 SQLite outbox + mirror。 离线优先，captures 毫秒级写进 L4 outbox；redactor 全程过滤每个 capture payload；MCP 暴露 L4 mirror，agent 读取零网络往返。
